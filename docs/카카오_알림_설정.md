# 4단계 — 카카오톡 에러 알림 설정 (로컬 · 초보용)

DB 장애·서버 오류 시 **본인 카카오톡「나에게 보내기」** 로 알림을 받습니다.

---

## 준비물

- 카카오 계정
- [Kakao Developers](https://developers.kakao.com/) 가입
- `.env` 파일 (프로젝트 루트, Git에 올리지 않음)

---

## ① Kakao Developers 앱 만들기

1. https://developers.kakao.com/ 로그인
2. **내 애플리케이션** → **애플리케이션 추가하기**
3. 앱 이름 입력 (예: `hanwhagreen-alert`) → 저장
4. **앱 키** → **REST API 키** 복사 → 메모장에 저장

---

## ② 카카오 로그인 + Redirect URI

1. 왼쪽 **카카오 로그인** → **활성화** ON
2. **Redirect URI** 등록:

```
http://localhost:8765/callback
```

3. **저장**

---

## ③ 동의 항목 (talk_message)

1. **카카오 로그인** → **동의항목**
2. **카카오톡 메시지 전송** (`talk_message`) → **필수** 또는 **선택** 동의 ON
3. 저장

---

## ③-2 클라이언트 시크릿 (ON 인 경우만)

**앱 키** → **REST API 키** → **클라이언트 시크릿** 이 **ON** 이면 `.env` 에 추가:

```
KAKAO_CLIENT_SECRET=여기에_클라이언트_시크릿
```

OFF 로 두면 이 항목은 생략해도 됩니다.

---

## ④ 리프레시 토큰 발급 (PC에서 1회)

프로젝트 폴더 PowerShell:

```powershell
cd c:\Users\psw00\OneDrive\Desktop\docker_exam2
copy .env.example .env
```

`.env` 에 REST API 키 입력:

```
KAKAO_REST_API_KEY=여기에_REST_API_키
```

토큰 발급 스크립트 실행:

```powershell
node scripts/kakao-setup-token.mjs
```

1. 브라우저 URL이 출력됨 → **카카오 로그인** → **동의**
2. 터미널에 **리프레시 토큰** 출력
3. `.env` 에 추가:

```
KAKAO_REFRESH_TOKEN=출력된_토큰
ADMIN_NOTIFY_ENABLED=1
NOTIFY_PROVIDER=kakao
```

---

## ⑤ Docker 재시작

```powershell
docker compose up -d --build
```

---

## ⑥ 테스트 (카카오만)

DB 중지:

```powershell
docker compose stop db
```

15~30초 후 카카오톡 **「나에게 보내기」** 확인.

복구:

```powershell
docker compose start db
```

---

## 카카오가 안 될 때 — 텔레그램 대체

1. 텔레그램 **@BotFather** → `/newbot` → 토큰 발급
2. 봇에게 `/start` 전송
3. 브라우저: `https://api.telegram.org/bot<토큰>/getUpdates` → `chat_id` 확인
4. `.env`:

```
NOTIFY_PROVIDER=telegram
TELEGRAM_BOT_TOKEN=...
TELEGRAM_CHAT_ID=...
```

## Playwright 데모 (브라우저 보면서)

```powershell
npm run test:e2e:db-error
```

브라우저가 열리며: 홈 구경 → DB 중지 → 로그인 실패 확인 → (카카오 알림) → DB 복구 → 로그인 성공.
