# 서버 삭제·복구 연습 (EC2, 초보용)

EC2에서 “사고가 났다”고 가정하고, Git + Docker로 홈페이지를 되살리는 연습입니다.

---

## 먼저 읽기

| 저장소 | 내용 | Git 복구 | 백업 복구 |
|--------|------|----------|-----------|
| GitHub | 소스 코드 | ✅ | — |
| Docker 볼륨 `postgres_data` | DB 데이터 | ❌ | ✅ `.sql` |
| `.env.ssl`, `ssl-active.conf` | HTTPS | ❌ | WinSCP로 수동 |

**연습 순서:** [DB_백업_복원_안내.md](DB_백업_복원_안내.md)로 **백업 먼저** → 아래 2-A → (선택) 2-B

---

## 0단계 — 사전 점검

```bash
cd ~/hanwhagreen_test
git pull origin main
chmod +x scripts/ec2-pre-check.sh
./scripts/ec2-pre-check.sh
```

결과 파일: `backups/pre-check_latest.txt`  
복구 후 같은 스크립트를 다시 실행해 **공지·QA 개수**를 비교하세요.

HTTPS 사용 시 WinSCP로 PC에 미리 복사:
- `.env.ssl`
- `nginx/ssl-active.conf`

---

## 2-A — 가벼운 사고 (코드만 삭제, DB 볼륨 유지)

### 상황

프로젝트 폴더를 지웠지만, **`down -v` 는 하지 않음** → DB 데이터는 Docker 볼륨에 남음.

### ① 사고 재현

```bash
cd ~/hanwhagreen_test
docker compose -f docker-compose.yml -f docker-compose.ec2.yml down
cd ~
rm -rf ~/hanwhagreen_test
```

브라우저 `http://퍼블릭IP/` → 접속 불가 확인.

### ② Git 복구

```bash
git clone https://github.com/parkseonguk125/hanwhagreen_test.git ~/hanwhagreen_test
cd ~/hanwhagreen_test
chmod +x scripts/ec2-deploy.sh

# HTTPS 쓰던 경우: .env.ssl, nginx/ssl-active.conf 다시 업로드

./scripts/ec2-deploy.sh
```

### ③ 확인

```bash
./scripts/ec2-pre-check.sh
diff backups/pre-check_latest.txt backups/pre-check_latest.txt
# (복구 전 파일을 다른 이름으로 저장해 두었다면 diff로 비교)
```

- 홈페이지 접속 OK
- **공지·QA 개수가 사고 전과 같으면** 성공 (볼륨이 살아 있었기 때문)

## 2-B — 심각한 사고 (볼륨 삭제 + 백업 복원)

> **반드시** `db-backup.sh`로 백업을 만든 **후** 진행하세요.

### ① 사고 재현

```bash
cd ~/hanwhagreen_test
docker compose -f docker-compose.yml -f docker-compose.ec2.yml down -v
cd ~
rm -rf ~/hanwhagreen_test
```

`-v` = DB 볼륨까지 삭제 → 게시글 데이터 전부 사라짐.

### ② Git으로 코드 복구 (2-A와 동일)

```bash
git clone https://github.com/parkseonguk125/hanwhagreen_test.git ~/hanwhagreen_test
cd ~/hanwhagreen_test
./scripts/ec2-deploy.sh
```

이 시점 DB = **초기 시드만** (공지 1건, QA 1건, admin).

### ③ 백업으로 DB 복구

백업 파일이 clone 전에 `~/` 등 다른 경로에 있어야 합니다.  
**팁:** 2-B 전에 WinSCP로 `backups/*.sql`을 PC에 받아 두고, clone 후 다시 업로드.

```bash
cd ~/hanwhagreen_test
./scripts/db-restore.sh backups/hanwhagreen_YYYYMMDD_HHMMSS.sql
```

### ④ 확인

```bash
./scripts/ec2-pre-check.sh
```

0단계에서 기록한 **공지·QA 개수**가 돌아왔는지 확인.

## 코드 수정 후 EC2 반영

로컬에서 GitHub push → EC2:

```bash
cd ~/hanwhagreen_test
./scripts/ec2-deploy.sh
```
