## S1 — 관찰 연습

### 사고 재현

```powershell
npm run local:security-s1
```

또는 `scripts/security-drill-s1.ps1` — 로그인 실패 3회 (임계치 미만)

### 확인

```powershell
docker compose logs api --tail 20
```

`[security]` 로그에 `login_failed` 가 보이면 성공.

### 조치

```bash
./scripts/security-incident-snapshot.sh
```

`backups/incident_*` 폴더에 로그·health 스냅샷이 생성되었는지 확인.

## S2 — 주의 연습

### 사전: 알림 설정

[카카오_알림_설정.md](카카오_알림_설정.md) — `.env`에 `ADMIN_NOTIFY_ENABLED=1`

API 재시작:

```powershell
docker compose up -d --build api
```

### 사고 재현

```powershell
npm run local:security-s2
```

로그인 실패를 rate limit(기본 15분 10회)까지 반복 → **429** 응답.

### 확인

| 확인 | 방법 |
|------|------|
| 429 | 스크립트 마지막 출력 또는 `curl -X POST http://localhost:8081/api/auth/login ...` |
| 알림 | 카카오/Telegram S2 메시지 |
| 로그 | `docker compose logs api --tail 30 \| findstr security` |

### 조치 (로컬)

로컬에서는 UFW 대신 **로그 IP 기록**만 합니다. EC2에서는 `security-block-ip.sh` 사용.

### 복구

15분 대기 후 rate limit 자동 해제, 또는:

```powershell
docker compose restart api
```
