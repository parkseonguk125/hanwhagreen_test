# 모의해킹 대비훈련 — 개요

「해킹을 당했다」고 가정하고, **심각도(S1~S4)**에 따라 탐지·기록·조치·복구를 연습합니다.

> **EC2 배포는 로컬 연습·E2E 검증이 끝난 뒤 마지막에만** 진행하세요. ([모의해킹_EC2_하드닝_및_조치.md](모의해킹_EC2_하드닝_및_조치.md))

## 심각도 한눈에 보기

| 심각도 | 명칭 | 대표 시나리오 | 1차 조치 |
|--------|------|---------------|----------|
| **S1** | 관찰 | 포트 스캔, 소량 로그인 실패 | 스냅샷·로그 기록 |
| **S2** | 주의 | 로그인 무차별 대입(임계치 초과) | rate limit, IP 차단, 알림 |
| **S3** | 경고 | 관리자 세션·데이터 변조 의심 | 유지보수 모드, 세션 무효화, 비밀번호 교체 |
| **S4** | 치명 | 서버 침투·DB 유출·서비스 파괴 | 격리, 증거 보존, 백업·EC2 재구축 |

## 연습 순서 (로컬 → EC2)

```
0. 사전 점검 (ec2-pre-check / security-incident-snapshot)
1. S1/S2 로컬 연습 → docs/모의해킹_연습_S1_S2_로컬.md
2. S3/S4 로컬 연습 → docs/모의해킹_연습_S3_S4_로컬.md
3. E2E 시연 → npm run test:e2e:security-s2
4. (마지막) EC2 하드닝·동일 연습 → docs/모의해킹_EC2_하드닝_및_조치.md
```

## 알림 설정

S2 이상 이벤트는 [카카오_알림_설정.md](카카오_알림_설정.md)와 동일한 `.env` 설정을 사용합니다.

```env
ADMIN_NOTIFY_ENABLED=1
NOTIFY_PROVIDER=kakao
NOTIFY_ENV_LABEL=local
SECURITY_NOTIFY_MIN_SEVERITY=2
```
