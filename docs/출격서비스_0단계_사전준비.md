# 출격 서비스 0단계 — 사전준비 (초보자용)

> **이 문서가 하는 일**: Flutter 출격 앱과 홈페이지 연동을 시작하기 **전에** 필요한 도구·폴더·API 키를 준비합니다.  
> **지금 완료된 것**: `mobile/` Flutter 프로젝트 생성, 설정 파일·점검 스크립트·안내 문서.  
> **아직 안 한 것**: 출격 API(1단계), 홈페이지 메뉴(2단계), 앱 본기능(3단계), APK 배포(4단계).

---

## 1. 전체 그림 (한 줄 요약)

| 무엇 | 어디 | 비유 |
|------|------|------|
| **홈페이지** | `frontend/` + `backend/` | 손님이 보는 카페 + 주방 |
| **출격 앱** | `mobile/` | 직원이 현장에서 쓰는 **주문 태블릿** |
| **연결 통로** | `/api/attendance` (1단계에서 만듦) | 태블릿 → 주방으로 주문 전달 |

앱과 웹은 **서로 다른 프로그램**이지만, 같은 **API 서버**(`backend`)에 데이터를 보냅니다.

---

## 2. Flutter란? (처음이신 분)

- **Google이 만든 앱 제작 도구**입니다.
- **Dart**라는 언어로 화면·버튼·카메라 등을 만듭니다.
- 한 번 만들면 **Android APK**로 배포할 수 있습니다 (이번 목표).
- 홈페이지는 **React**, 앱은 **Flutter** — 역할만 같고 기술은 다릅니다.

### 자주 나오는 단어

| 용어 | 뜻 |
|------|-----|
| **SDK** | 앱/프로그램을 만들 때 쓰는 도구 모음 (Flutter SDK) |
| **APK** | Android 설치 파일. 카카오톡으로 보내서 설치 가능 |
| **dart-define** | 앱 빌드·실행할 때 넣는 **설정값** (API 주소, 비밀키 등) |
| **Hot Reload** | 코드 저장하면 앱이 거의 즉시 갱신 (개발 중 편리) |

## 4. Flutter SDK (이 PC에 설치된 방식)

전역 설치 대신, 프로젝트 안 **`.tools/flutter/`** 에 SDK를 받아 두었습니다.

- **장점**: PATH 설정 없이 스크립트로 바로 사용
- **단점**: 용량 약 1GB+ → **Git에는 올리지 않음** (`.gitignore` 처리)

### SDK가 없을 때 (다른 PC에서 clone 후)

PowerShell에서 프로젝트 루트에서:

```powershell
git clone https://github.com/flutter/flutter.git -b stable --depth 1 .tools/flutter
```

## 5. Android Studio (APK 빌드에 필요)

`flutter doctor` 결과 **Android toolchain** 이 X 이면 APK를 아직 못 만듭니다. **Chrome으로는** 0단계 화면 테스트 가능.

### 설치 순서

1. [Android Studio](https://developer.android.com/studio) 다운로드·설치
2. 첫 실행 → **Android SDK** 설치 마법사 따라하기
3. 터미널에서 다시 점검:

```powershell
.\mobile\scripts\check-setup.ps1
```

또는:

```powershell
.\mobile\scripts\flutter.ps1 doctor
```

모든 Android 관련 항목이 ✓ 이면 APK 빌드 준비 완료입니다.

## 6. API 키·환경변수 준비

### 6-1. 서버용 — `.env`

프로젝트 루트 `.env.example` 을 복사해 `.env` 를 만든 뒤 아래를 채웁니다.

```env
# 출격 앱 → 서버 제출 인증 (랜덤 긴 문자열, 예: openssl rand -hex 32)
ATTENDANCE_APP_API_KEY=여기에_랜덤_문자열

# 출격 사진 저장 폴더 (Docker 기본값과 동일하게 두면 됨)
# ATTENDANCE_UPLOAD_DIR=/app/uploads/attendance
```

**왜 필요한가?**  
앱만 APK로 배포하면 누구나 설치할 수 있습니다. 서버는 **`X-App-Key` 헤더**가 `.env` 값과 같을 때만 출격 기록을 받습니다 (사내용 중간 보안).

### 6-2. 앱용 — `--dart-define`

앱에는 `.env` 파일을 넣지 않고, 실행·빌드할 때 값을 **주입**합니다.

| dart-define 이름 | .env 대응 | 설명 |
|------------------|-----------|------|
| `API_BASE_URL` | — | API 주소, 예: `http://localhost:3001/api` |
| `APP_API_KEY` | `ATTENDANCE_APP_API_KEY` | **동일한 값** |
| `NAVER_MAP_CLIENT_ID` | — | NCP **Mobile** 지도 Client ID |

예시 파일: [`mobile/config/dart-defines.example.txt`](../mobile/config/dart-defines.example.txt)

## 7. 네이버 지도 Client ID (웹과 **별도**)

홈페이지는 **Web Dynamic Map** (`VITE_NAVER_MAP_CLIENT_ID`) 을 씁니다.  
**앱**은 **Mobile SDK** 용 Client ID가 **따로** 필요합니다.

### 발급 절차

1. [NCP 콘솔](https://console.ncloud.com) 로그인
2. **AI·NAVER API → Application** → 앱 등록 (또는 기존 앱 수정)
3. **Mobile Dynamic Map** (또는 Mobile SDK) 선택
4. Android 패키지명 등록: `com.hanwhagreen.hanwha_attendance`
5. 발급된 **Client ID** → `NAVER_MAP_CLIENT_ID` 로 앱에 넣기

> 3단계에서 지도 화면을 만들 때 이 키를 사용합니다. 0단계에서는 **발급만** 해 두면 됩니다.

### 방법 C — 앱 화면 (Chrome)

Android Studio 없이도 **웹 브라우저**로 0단계 화면을 볼 수 있습니다.

```powershell
cd mobile
..\mobile\scripts\flutter.ps1 run -d chrome
```

키를 넣어 테스트하려면:

```powershell
..\mobile\scripts\flutter.ps1 run -d chrome `
  --dart-define=API_BASE_URL=http://localhost:3001/api `
  --dart-define=APP_API_KEY=테스트용키 `
  --dart-define=NAVER_MAP_CLIENT_ID=모바일_Client_ID
```

화면에 **초록 체크 / 주황 안내** 카드가 보이면 Flutter 프로젝트는 정상입니다.

## 11. 문제 해결

| 증상 | 해결 |
|------|------|
| `'flutter' is not recognized` | `.\mobile\scripts\flutter.ps1` 사용 (전역 PATH 불필요) |
| Android toolchain X | Android Studio 설치 후 SDK 설치 |
| Chrome run은 되는데 폰 연결 안 됨 | USB 디버깅·드라이버 확인, `flutter devices` |
| 실기기에서 API 연결 실패 | `localhost` 대신 **PC IP** 또는 **배포 도메인** 사용 |
| `.tools/flutter` 용량 | 정상. Git push 시 제외됨 |
