# 모의해킹 연습 — S3/S4 (로컬 Docker)

S3는 **의도적으로 서비스를 잠시 끕니다.** 연습 후 반드시 복구하세요.

---

## S3 — 경고 연습

### 0. 사전 점검

```bash
./scripts/ec2-pre-check.sh
```

(로컬에서도 동일 스크립트 사용 가능 — DB 컨테이너명 `hanwhagreen-db` 기준)

### 1. 스냅샷

```bash
./scripts/security-incident-snapshot.sh
```

### 2. 유지보수 모드 ON

```bash
./scripts/security-maintenance-on.sh
```

브라우저 `http://localhost:8081` → **점검 중** 페이지 확인 (유지보수 HTML).

### 3. 세션 무효화

```bash
./scripts/security-invalidate-sessions.sh
```

### 4. admin 비밀번호 변경

```bash
./scripts/security-rotate-admin-password.sh
```

새 비밀번호를 안전한 곳에 메모하세요.

### 5. DB 백업

```bash
./scripts/db-backup.sh
```

### 6. 복구

```bash
./scripts/security-maintenance-off.sh
```

새 비밀번호로 `/bbs/login.php` 로그인 확인.

### 7. 사후 점검

```bash
./scripts/ec2-pre-check.sh
```

공지·QA 개수가 사고 전과 같은지 확인.

## S4 — 치명 연습 (로컬 시뮬레이션)

실제 EC2 terminate는 하지 않습니다. **서비스 격리 + 백업 복원**만 연습합니다.

### 1. 격리

```bash
./scripts/security-isolate.sh
```

홈페이지 접속 불가 확인.

### 2. 스냅샷 (가능하면 격리 전에 실행 권장)

```bash
./scripts/security-incident-snapshot.sh
```

### 3. 서비스 재기동

```bash
docker compose up -d
```

### 4. DB 복원 연습 (선택)

[DB_백업_복원_안내.md](DB_백업_복원_안내.md) — 방금 만든 `.sql`로 `db-restore.sh` 실행.

### 5. EC2 최악 시나리오

실제 2-C 연습은 [서버_삭제_복구_연습_안내.md](서버_삭제_복구_연습_안내.md) — **로컬·EC2 모두 검증 완료 후** 진행.
