# 모의해킹 — 심각도 분류 및 조치표

## S1 — 관찰

| 항목 | 내용 |
|------|------|
| **상황** | 포트 스캔, `/api/auth/login` 소량 실패(임계치 미만), 비정상 User-Agent |
| **영향** | 서비스 정상, 의심 활동만 기록 |
| **탐지** | Nginx access log, API `[security]` 로그 |
| **조치** | `security-incident-snapshot.sh` 실행, 로그 보관, 모니터링 지속 |
| **복구** | 별도 복구 없음 |

**체크리스트**

- [ ] `backups/incident_*` 스냅샷 생성
- [ ] 실패 IP·시각 기록

## S2 — 주의

| 항목 | 내용 |
|------|------|
| **상황** | 로그인 무차별 대입( rate limit 임계치 초과 → 429 ), 관리자 API 무인증 반복 |
| **영향** | 해당 IP 로그인 차단, 서비스는 유지 |
| **탐지** | rate limit, `notifySecurityEvent` (카카오/Telegram) |
| **조치** | `./scripts/security-block-ip.sh <IP>` (EC2), 로그 분석 |
| **복구** | 차단 해제: `security-unblock-ip.sh`, 또는 15분 후 rate limit 자동 해제 |
| **알림** | S2 — `SECURITY_NOTIFY_MIN_SEVERITY=2` 이상 시 자동 |

**체크리스트**

- [ ] 알림 수신 확인
- [ ] `docker compose logs api --tail 100 | grep security`
- [ ] 공격 IP 차단 여부 결정
- [ ] 정상 사용자 오탐 없는지 확인

## S3 — 경고

| 항목 | 내용 |
|------|------|
| **상황** | 관리자 세션 탈취 의심, 공지/QA 무단 변조, 악성 업로드 시도 |
| **영향** | 일시적 서비스 중단 가능, 전원 재로그인 |

| **알림** | 수동 + S2 알림 체계 활용 |

**체크리스트**

- [ ] `security-maintenance-on.sh` — 외부 503 확인

- [ ] `db-backup.sh` 즉시 실행

## 대응 흐름

```
탐지 → 심각도 분류 → 스냅샷 → 조치 → 확인 → 복구 → 사후 점검(ec2-pre-check)
```

| 단계 | 담당(연습 시) | 기록 위치 |
|------|---------------|-----------|
| 탐지 | 운영자 | API 로그, 알림 |
| 분류 | 운영자 | incident 스냅샷 메모 |
| 조치 | 운영자 | 이 문서 체크리스트 |
| 복구 | 운영자 | `backups/pre-check_latest.txt` 비교 |
