# 7번 — 도메인 연결 (무료 · 초보용)

EC2 IP(`http://13.209.17.85/`) 대신 **이름 주소**로 접속하게 만드는 단계입니다.

---

## 시작 전에 알아두기

| 항목 | 설명 |
|------|------|
| **완전 무료 `.com`** | `hanwhagreen_test.com` 같은 **.com 은 거의 항상 유료**(연 1만 원대). “100% 무료 .com” 은 실습에 비추천 |
| **도메인에 `_` (밑줄)** | **사용 불가** → `hanwhagreen-test` 처럼 **하이픈(-)** 사용 |
| **이 가이드 추천** | **[DuckDNS](https://www.duckdns.org/)** — 무료, 가입 간단, 현업에서도 **개발·테스트 URL** 로 자주 씀 |
| **접속 예시** | `http://hanwhagreentest.duckdns.org/` (본인이 정한 이름) |
| **EC2** | **실행 중** + **80번(HTTP)** 열려 있어야 함 (5~6번 완료 상태) |
| **현재 IP** | `13.209.17.85` (인스턴스 **중지 후 재시작**하면 바뀔 수 있음 → 아래 **IP 바뀔 때** 참고) |

---

## 전체 순서 (한눈에)

```
① DuckDNS 가입 (Google 등)
② 원하는 이름 만들기 (예: hanwhagreentest)
③ IP에 13.209.17.85 넣고 저장
④ 5~10분 기다리기
⑤ 브라우저 http://이름.duckdns.org/ 확인
```

**서버(EC2)에 추가 설치할 것 없음** — DNS만 IP를 가리키면 됩니다.

---

## ① DuckDNS 가입

1. 브라우저에서 **https://www.duckdns.org/** 접속  
2. **sign in with google** (또는 GitHub 등) 클릭 → 로그인  
3. 로그인되면 **domains** 페이지가 보임  

---

## ② 무료 주소(서브도메인) 만들기

화면에 **sub domain** 입력 칸이 있습니다.

1. 원하는 이름 입력 (영문·숫자, **밑줄 없음**)  
   - 예: `hanwhagreentest`  
   - 결과 주소: **`hanwhagreentest.duckdns.org`**
2. **add domain** 클릭  
3. 목록에 `hanwhagreentest.duckdns.org` 가 추가되면 성공  

> 이미 다른 사람이 쓰는 이름이면 **다른 이름**을 시도하세요.  
> 예: `psw-hanwhagreen`, `hanwhagreen2026` 등

---

## ③ EC2 IP 연결 (가장 중요)

같은 줄(도메인)에 **current ip** 또는 IP 입력 칸이 있습니다.

1. IP 칸에 **퍼블릭 IPv4** 입력  
   ```
   13.209.17.85
   ```
2. **update ip** (또는 갱신 버튼) 클릭  
3. 화면에 IP가 `13.209.17.85` 로 보이면 OK  

### IP 확인 방법 (AWS)

1. AWS 콘솔 → **EC2** → **인스턴스**  
2. 인스턴스 클릭  
3. 아래 **퍼블릭 IPv4 주소** 복사 → DuckDNS에 붙여넣기  

---

## ④ 전파 대기

- 보통 **1~10분** (길면 30분)  
- 이 동안 EC2는 **실행 중**으로 두기  

---

## ⑤ 접속 확인 (7번 완료)

PC·휴대폰 브라우저 주소창에 (**https 아님**):

```
http://hanwhagreentest.duckdns.org/
```

(②에서 정한 이름으로 바꾸세요.)

| 확인 | 성공 |
|------|------|
| 한화그린 홈 화면 | ✅ |
| `http://13.209.17.85/` 와 **같은 사이트** | ✅ |
| 공지·온라인문의 열림 | ✅ (선택) |

**7번 완료** = IP 대신 **이름으로** 접속되는 것.

---

## IP가 바뀌었을 때 (중지 후 재시작 등)

1. AWS에서 **새 퍼블릭 IP** 확인  
2. DuckDNS → 해당 도메인 → IP 수정 → **update ip**  
3. 몇 분 후 새 IP 또는 도메인으로 접속  

### IP 고정 (선택, AWS)

- **탄력 IP(Elastic IP)** 를 EC2에 연결하면 IP가 안 바뀜 (과금·프리티어 규칙 확인)  
- 도메인은 **탄력 IP**를 가리키게 설정  

---

## (참고) 0원에 가깝게 “IP만” 이름 쓰기 — sslip.io

설정 없이 **즉시** 테스트만 할 때:

```
http://13-209-17-85.sslip.io/
```

- 무료, 설정 없음  
- **과제 제출용 고정 주소**로는 DuckDNS가 더 낫습니다 (이름을 직접 정할 수 있음)  

---

## (나중에) 유료 `.com` — 현업에서 많이 씀

실제 서비스·포트폴리오용 **`.com`** 이 필요하면:

| 서비스 | 특징 |
|--------|------|
| [Cloudflare Registrar](https://www.cloudflare.com/products/registrar/) | DNS·등록 한곳, 현업에서 매우 흔함 |
| AWS Route 53 | AWS와 함께 쓸 때 많음 |
| 가비아 / Namecheap | 국내·해외 등록 업체 |

절차: 도메인 구매 → DNS **A 레코드** `@` → `13.209.17.85` → (선택) HTTPS

## 자주 묻는 것

**Q. `hanwhagreen_test.com` 은 안 되나요?**  
A. `.com` 은 **구매**가 필요하고, 이름에 **`_` 는 불가**합니다. 무료로는 `hanwhagreentest.duckdns.org` 형식을 쓰세요.

**Q. DuckDNS 말고 완전 무료 .com 있나요?**  
A. 신뢰할 만한 **무료 .com** 은 거의 없습니다. 실습은 DuckDNS, 상용은 유료 `.com` 을 권장합니다.

**Q. https(자물쇠)는?**  
A. 7번 기본은 **http**. HTTPS는 Let’s Encrypt·Cloudflare SSL 등 **추가 작업**입니다.

**Q. PC 꺼도 도메인 되나요?**  
A. **EC2만 실행 중**이면 됩니다. PC·SSH 창과 무관합니다.
