# 프로젝트 구조 안내 (초보자용)

이 문서는 **어떤 폴더가 무슨 일을 하는지**, **파일끼리 어떻게 연결되는지** 정리한 것입니다.

---

## 1. 전체 그림

```
[브라우저]  →  web (Nginx, 포트 8081)
                 ├─ 정적 파일: frontend 빌드 결과 (HTML/JS/CSS)
                 └─ /api/*  →  api (Node.js, 포트 3000)
                                  └─ db (PostgreSQL)
```

| 역할 | 폴더 | 비유 |
|------|------|------|
| **화면** | `frontend/` | 손님이 보는 메뉴판·인테리어 |
| **서버** | `backend/` | 주방 (주문 받고 DB에 기록) |
| **DB** | `db/` + PostgreSQL 데이터 | 창고 (글·비밀번호 저장) |
| **실행 설계도** | `docker-compose.yml` | 세 공간을 한 번에 켜는 스위치 |
| **출격 앱** | `mobile/` | 현장 직원용 Flutter Android 앱 (APK) |

```
[Flutter APK]  →  /api/attendance  →  api → db   (1단계 이후)
[브라우저]    →  web → /api/attendance (출결서비스 목록·관리자 상세)
```

## 3. `frontend/` — 화면

### 3-1. 폴더 역할

| 폴더 | 역할 |
|------|------|
| `src/pages/` | **페이지** (URL 한 장당 하나) |
| `src/components/` | 페이지 안 **조각** (헤더, 푸터, 슬라이드…) |
| `src/components/board/` | **게시판 전용** 조각 |
| `src/services/` | **API 호출** (서버에 글 요청) |
| `src/styles/` | **CSS** (색·간격·게시판 모양) |
| `src/utils/` | 공통 **도구** (경로 만들기 등) |
| `src/data/` | 예시/임시 **mock 데이터** |
| `dist/` | 빌드 결과 (**직접 수정 X**, Docker가 사용) |

### 3-2. pages — 주요 화면

| 파일 | 화면 | URL 예시 |
|------|------|----------|
| `App.jsx` | 모든 페이지 **연결표** | — |
| `main.jsx` | React **시작** | — |
| `HomePage.jsx` | 메인 홈 | `/` |
| `NoticeBoardPage.jsx` | 공지사항 | `/bbs/board.php?bo_table=notice` |
| `QaBoardPage.jsx` | 온라인문의 목록 | `bo_table=qa` |
| `BoardWritePage.jsx` | 글쓰기·수정 | `/bbs/write.php` |
| `BoardPasswordPage.jsx` | 비밀글 비밀번호 | `/bbs/password.php` |
| `BoardRouter.jsx` | 공지/QA **분기** | `/bbs/board.php` |
| `LoginPage.jsx` | 로그인 (예정) | `/bbs/login.php` |

### 3-3. services — 서버와 통신

| 파일 | 설명 |
|------|------|
| `boardApi.js` | 공지·QA **CRUD** API 호출 |
| `boardAccess.js` | 비밀번호·열람 권한 **sessionStorage** |
| `authApi.js` | 회원 로그인 API (`/api/auth`) |
| `authAccess.js` | 로그인 토큰·회원 정보 저장 |
| `boardStorage.js` | 예전 로컬 저장 방식 (참고용) |

### 3-4. 게시판 관련 연결 (한 세트)

**온라인문의 글쓰기**

```
BoardWritePage.jsx  →  boardApi.js  →  (nginx)  →  backend/routes/qa.js  →  db.js
```

**공지사항 목록**

```
NoticeBoardPage.jsx  →  NoticeBoardView.jsx  →  boardApi.js  →  routes/notice.js
```

**스타일 (같이 보면 좋음)**

| 화면 | JSX | CSS |
|------|-----|-----|
| 글쓰기 | `BoardWritePage.jsx` | `qna-skin.css`, `board-pages.css` |
| 공지 목록 | `NoticeBoardView.jsx` | `notice-skin.css` |
| 비밀번호 | `BoardPasswordPage.jsx` | `password-page.css` |

### 3-5. 확장자

| 확장자 | 의미 |
|--------|------|
| `.jsx` | 화면 + 동작 (React) |
| `.js` | 로직·API |
| `.css` | 디자인만 |
| `.module.css` | 해당 컴포넌트만 쓰는 CSS |

## 4. `backend/` — API 서버

| 파일 | 설명 |
|------|------|
| `src/server.js` | 서버 시작, `/api/health`, 라우트 등록 |
| `src/db.js` | PostgreSQL 연결, 테이블, 글 저장·조회 |
| `src/routes/notice.js` | `/api/notice` 공지 API |
| `src/routes/qa.js` | `/api/qa` 온라인문의 API |
| `package.json` | 사용 라이브러리 목록 |

**`node_modules/`** — npm이 설치한 패키지. **수정·삭제하지 않음.**

---

## 5. `db/` — 데이터베이스 컨테이너

| 파일 | 설명 |
|------|------|
| `Dockerfile` | PostgreSQL 포함 이미지 |
| `docker-entrypoint.sh` | DB 컨테이너 **첫 실행** 스크립트 |

DB 안 테이블:

| 테이블 | 내용 |
|--------|------|
| `notice_posts` | 공지사항 글 |
| `qa_posts` | 온라인문의 글 (비밀번호 해시 포함) |
| `members` | 회원 (아이디, 비밀번호 해시, 이름, 권한) |
| `member_sessions` | 로그인 세션 토큰 |

## 7. 영어 이름 자주 쓰는 것

| 영어 | 한국어 뜻 |
|------|-----------|
| frontend | 앞단, 브라우저 화면 |
| backend / api | 뒷단, 서버 |
| compose | 여러 Docker를 한꺼번에 |
| page | 화면 한 페이지 |
| component | UI 조각 |
| service | API 호출 등 공통 기능 |
| route / routes | URL 처리 |
| board | 게시판 |
| notice | 공지 |
| qa / qna | 문의 |
| write | 글쓰기 |
| mock | 가짜 테스트 데이터 |
| dist | 빌드 결과물 |
| health | 서버 살아있는지 확인 |

---

## 8. 안 봐도 되는 것

| 경로 | 이유 |
|------|------|
| `backend/node_modules/` | 라이브러리 수천 개 |
| `frontend/node_modules/` | (있다면) 동일 |
| `frontend/dist/` | 자동 생성, 소스는 `src`만 수정 |

---

## 9. 수정할 때 어디를 보나

| 하고 싶은 일 | 주로 보는 곳 |
|--------------|--------------|
| 홈·배너 문구 | `frontend/src/pages/HomePage.jsx`, `components/` |
| 공지/문의 목록·글쓰기 | `frontend/src/pages/`, `components/board/` |
| API 동작·DB | `backend/src/db.js`, `routes/` |
| 사이트 안 뜸 / API 502 | `docker-compose.yml`, `nginx/` |
| DB 데이터 확인 | `docs/DB확인_안내.md`, `DB확인.bat` |
